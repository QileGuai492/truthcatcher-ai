# 实时新闻真伪辨别工具 — 项目大纲

## 一、项目概述

基于大语言模型（LLM）的实时新闻真伪辨别工具。用户输入新闻链接或文本，系统自动从网络采集足够信息并进行甄别分析，输出：新闻核心内容摘要与来龙去脉、真实性概率评分、支撑论据与推理过程。

**三大核心原则**：
1. **信息充分**：分析前必须从网络获取足够多的相关信息，不依赖模型内部知识做判断
2. **去伪存真**：对网络信息进行可信度筛选，识别并剔除虚假/谣言信息，仅以可靠信源作为论据
3. **公正客观**：不带立场、不夹带私货，全面呈现正反两面的事实与观点，让用户自行形成判断

## 二、核心功能模块

### 1. 新闻采集模块

- **URL 抓取**：支持用户粘贴新闻链接，自动提取正文内容（绕过反爬、广告、无关推荐）
- **文本输入**：支持直接粘贴新闻文本
- **截图/图片识别**（可选）：OCR 提取图片中的新闻文字

### 2. 网络信息检索模块

- **多搜索引擎并发**：同时调用多个搜索引擎（Google/Bing/百度等），最大化信息覆盖面
- **检索关键词自动生成**：由 LLM 根据新闻内容自动提取多个检索维度（事件关键方、核心主张、相关地点时间等）
- **结果去重与排序**：对搜索结果去重，按时效性和相关性排序
- **深度抓取**：对高价值页面进行正文提取，获取完整内容而非仅摘要

### 3. 信息可信度筛选模块

- **信源评级**：对每条网络信息标注来源类型（官方机构/权威媒体/自媒体/论坛/未知），赋予不同可信度权重
- **交叉验证机制**：同一事实主张需至少被两个独立信源确认，单一信源报道自动标记为"待验证"
- **虚假信号识别**：检测标题党、情绪化表述、逻辑矛盾、与已知事实冲突等虚假信息特征
- **时效性检查**：确认信息的发布时间，排除过时信息对当前判断的干扰

### 4. AI 分析引擎

- **来龙去脉梳理**：还原新闻事件的前因后果与完整背景，让用户了解全貌
- **摘要提取**：用 LLM 提炼新闻的核心事实主张（5W1H：谁、何时、何地、何事、为何、如何）
- **真伪判断**：基于网络采集的可靠信息与多源交叉验证结果，给出真实性概率（0%~100%）
- **论据生成**：逐条列出支撑判断的正反论据，每条标注来源链接
- **论证推理**：展示从论据到结论的完整逻辑链条，确保推理过程公开透明

### 5. 公平性保障模块

- **正反立场均衡**：强制要求模型同时搜集支持与反对新闻主张的证据，两边论据数量尽量对等
- **立场声明机制**：当某条论据的来源存在已知立场倾向时，明确标注该来源的政治/商业/文化背景
- **利益冲突检测**：识别并标注信源与新闻当事人之间可能存在的利益关联
- **不确定性透明化**：证据不足或矛盾时，如实呈现各方说法的分歧，而非强行判定真伪

### 6. 结果展示模块

- **真实性仪表盘**：圆形进度条展示真实概率，颜色从红（假）到绿（真）
- **论点卡片**：逐条展示论据，每条标注来源和可信度
- **推理可视化**：以流程图或思维导图呈现从论据到结论的推理路径
- **原文高亮**：在原文中标注可疑片段和可信片段

## 三、技术架构（建议）

```text
                         ┌──────────────┐
                         │   搜索引擎集群  │
                         │ Google/Bing/百度│
                         └──────┬───────┘
                                │
┌─────────────┐    ┌────────────┼──────┐    ┌─────────────┐
│  前端界面    │───>│  后端 API   │      │───>│  AI 服务层   │
│  (Web/GUI)  │<───│  (Python)   │<─────┘    │  (LLM API)  │
└─────────────┘    └──────┬──────┘           └──────┬──────┘
                          │                         │
                    ┌─────┴──────┐          ┌──────┴──────┐
                    │  爬虫 +     │          │  Prompt 模板 │
                    │  正文提取    │          │  公正性约束   │
                    └────────────┘          └─────────────┘
```

- **前端**：React / Vue 或 Python GUI（如 PyQt、Streamlit）
- **后端**：Python FastAPI / Flask
- **AI 服务**：调用 Claude API / OpenAI API，配合精心设计的公正性 prompt 模板
- **搜索**：SerpAPI / Bing API / Google Custom Search，多引擎并发
- **爬虫**：BeautifulSoup / Playwright 进行正文提取，绕过反爬
- **数据持久化**：SQLite / PostgreSQL 存储历史分析记录和信源评级数据

## 四、输出内容规范

每条新闻分析结果包含以下五个必选项：

| 字段 | 说明 | 示例 |
| ---- | ---- | ---- |
| **来龙去脉** | 还原事件的前因后果和完整背景，帮助用户全面了解新闻全貌 | "该说法源于某自媒体账号于X月X日的发帖，此前该账号曾多次发布类似未经证实的消息……" |
| **新闻摘要** | 200字以内概括核心事实主张 | "某地声称将于2030年前建成太空电梯……" |
| **真实概率** | 0~100% 的量化评估，附带置信度说明 | 真实概率：12%（置信度：高，因多源交叉验证结果一致） |
| **论据列表** | 4~10条正反论据，每条标注倾向、来源链接、信源可信度评级 | 【反-高可信】该技术目前无可行材料方案（来源：Nature论文）；【反-中信】信源仅有一家自媒体…… |
| **论证过程** | 完整逻辑推理链，说明如何从论据加权得出最终结论，明确标注推理中每一步的确定性程度 | "综合8条论据（5条反向/3条正向），反向论据均来自高可信信源且结论一致，正向论据均来自单一低信源，故判定为大概率虚假" |

## 五、Prompt 设计要点

AI 分析 prompt 需包含以下指令要素：

1. **角色设定**：你是一名中立、严谨的新闻事实核查员，你的目标不是说服用户相信某个结论，而是提供全面、可靠的信息帮助用户自行判断
2. **信息来源要求**：必须基于网络搜索结果进行分析，不得仅依赖模型训练数据；每个论据必须附带可追溯的公开来源链接
3. **信息筛选要求**：优先采信官方机构、权威媒体、学术论文等高质量信源；对单一信源、匿名来源、利益相关方的信息明确标注可信度等级
4. **公正性要求**：必须同时呈现支持和反对新闻主张的证据；当存在多种合理观点时，平等展示各方立场，不偏袒任何一方
5. **输出格式约束**：严格按 JSON/结构化格式输出五部分内容（来龙去脉、摘要、概率、论据、论证）
6. **推理要求**：必须展示完整的逻辑链条，不得跳过中间步骤；推理中每步需标注确定性程度
7. **不确定性表达**：证据不足时坦诚说明当前信息缺口，而非强行判断；结论需附带置信度
8. **偏见警示**：当论据来源存在已知的立场倾向或利益关联时，必须向用户明确标注

## 六、潜在扩展方向

- **浏览器插件**：浏览新闻时一键检测
- **谣言数据库**：建立已知谣言特征库，加速匹配
- **传播路径追踪**：展示新闻在各平台的传播图谱
- **多语言支持**：中英文新闻交叉验证
- **用户反馈闭环**：用户可对分析结果评分，持续优化模型判断

## 七、开发路线图

- **Phase 1**：MVP — 命令行工具，输入文本 → 自动搜索引擎检索 → LLM 综合分析 → 输出五部分结构化结果
- **Phase 2**：Web 界面 + URL 自动抓取 + 搜索引擎集成 + 信源可信度评级
- **Phase 3**：多源交叉验证 + 虚假信息特征识别 + 传播路径追踪 + 公正性保障机制
- **Phase 4**：浏览器插件 + 谣言数据库 + 用户反馈闭环
