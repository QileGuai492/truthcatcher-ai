# 项目计划书 — AI 实时新闻真伪辨别平台「真相捕手」

---

## 一、项目概述

### 1.1 项目名称

**真相捕手（TruthCatcher）** —— AI 驱动的实时新闻真伪辨别平台

### 1.2 所属行业

人工智能 / 信息服务业 / 媒体科技（MediaTech）

### 1.3 项目背景

虚假信息在社交媒体上的传播速度是真相的6倍。生成式AI进一步降低了虚假内容的生产成本。现有的人工核查平台每天只能处理3-5条新闻，远不能满足实时甄别需求。

### 1.4 核心产品或服务

用户通过网页端或浏览器插件输入新闻链接/文本，系统自动完成：

1. **全网信息检索** — SerpAPI（Google）搜索，中英文关键词并发
2. **信源可信度筛选** — A/B/C/D 四级评级 + 多源交叉验证 + 虚假信号识别
3. **AI 综合分析** — DeepSeek V4-Flash 推理，核心主张加权评分
4. **结构化输出** — 来龙去脉、摘要、真实概率、正反论据、论证过程

### 1.5 项目愿景

成为华语互联网用户最信赖的第三方新闻事实核查工具。

## 二、市场分析

### 2.1 目标用户画像

| 用户群体 | 占比估计 | 使用场景 | 核心诉求 |
| -------- | -------- | -------- | -------- |
| **普通网民** | 60% | 看到朋友圈/微博/抖音转发的惊人消息后想验证真假 | 操作简单、结论直观、免费使用 |
| **内容创作者/自媒体** | 20% | 选题前核实信源，避免引用虚假信息 | 来源可追溯、论据充分 |
| **企业公关/风控部门** | 12% | 监测涉及本企业的舆情真伪 | 实时预警、批量监测、专业报告 |
| **学术/研究机构** | 5% | 研究虚假信息传播规律 | 数据导出、传播路径可视化 |
| **政府/公共机构** | 3% | 舆情监控、谣言治理 | 高准确率、数据本地化 |

### 2.2 竞争对手分析

| 竞争对手 | 类型 | 优势 | 劣势 |
| -------- | ---- | ---- | ---- |
| **澎湃·明查** | 人工核查 | 专业编辑、权威性强 | 每天3-5条，无法实时响应 |
| **腾讯较真** | 平台自建 | 腾讯生态、用户基数大 | 仅核实特定栏目，不开放用户自主查询 |
| **Snopes.com** | 国际人工核查 | 历史悠久、覆盖谣言 | 人工瓶颈、中文内容极少 |
| **NewsGuard** | 信源评级 | 7000+网站评级、浏览器插件 | 仅评级信源，不分析具体内容 |
| **通用AI工具** | AI对话 | 即时响应 | 无系统化网络检索验证流程 |

### 2.3 差异化竞争优势

1. **全自动核查流水线** — 检索→筛选→验证→分析→输出，<30秒完成
2. **加权真实度评分** — 核心主张拆分加权，支持论据与反对论据强制均衡
3. **多源交叉验证** — 每条论据标注"已交叉验证"或"单一信源"
4. **传播路径追踪** — 首发来源 + 跨平台传播时间线
5. **中立透明** — 标注信源立场倾向和利益关联，不做一槌定音
6. **谣言数据库** — 已知谣言秒级匹配，新谣言自动积累

## 三、产品或服务

### 3.1 产品现状

**MVP 已完成**。Web 界面 + 浏览器插件均已可用，所有 Phase 1-4 功能已实现。

### 3.2 已实现功能清单

**用户端**：
- 文本/URL 输入分析，流式进度展示
- 概率仪表盘 + 核心主张进度条 + 论据卡片 + 传播路径时间线
- 论据逐条溯源（来源链接 + 信源等级 + 交叉验证标签）
- 可疑信号红色预警 + 谣言匹配提示
- 分析历史记录（下拉框回顾）
- 一键下载分析报告（TXT）
- Edge/Chrome 浏览器插件（选中页面文字一键分析）

**后端**：
- SerpAPI 搜索 + 中英文关键词自动生成
- 网页正文提取（httpx → Jina Reader → Playwright 三级兜底）
- 信源 A/B/C/D 四级自动评级（100+ 域名）
- 多源交叉验证（cross_verified + verified_by）
- 虚假信号检测（标题党/情绪/信源缺失/逻辑矛盾/利益导向）
- 核心主张加权评分（AI 分配权重，加权平均）
- 谣言数据库（模糊匹配 + 自动入库）
- 结果缓存（5 分钟 TTL）
- 多 Provider 支持（DeepSeek / Anthropic / OpenAI 兼容）

### 3.3 产品亮点

**亮点一：核心主张加权评分**

不同于单一概率值，将新闻拆解为 3-5 条核心主张，AI 为每条打分并分配权重，加权平均得出总概率。用户能看到新闻中哪些部分可信、哪些部分存疑。

**亮点二：信源评级 + 交叉验证**

每条论据标注来源类型和 A/B/C/D 等级，同一事实需 ≥2 个独立信源确认才算"已交叉验证"。

**亮点三：传播路径追踪**

展示新闻的首发平台、时间，以及跨平台传播的关键节点。

**亮点四：谣言数据库自积累**

每次判定为假的新闻自动入库。再次遇到相同或相似内容时秒出结果，无需重复分析。

## 四、商业模式

### 4.1 收入模型

| 层级 | 定价 | 目标用户 | 核心权益 |
| ---- | ---- | -------- | -------- |
| **免费版** | ¥0/月 | 普通网民 | 每日3次分析、文本输入、基础结果 |
| **专业版** | ¥29/月 | 内容创作者 | 每日50次、URL输入、论据溯源、历史记录、报告导出 |
| **企业版** | ¥299/月起 | 企业/机构 | 批量监测、API接口、定制信源权重 |
| **定制版** | 议价 | 政府/学术 | 本地化部署、模型微调、专属格式 |

### 4.2 成本结构（起步阶段）

| 成本项 | 月均预估 |
| ------ | -------- |
| LLM API（DeepSeek V4-Flash）| ¥200-500 |
| SerpAPI 搜索 | ¥100-300 |
| 云服务器 | ¥100-200 |
| **合计** | **¥400-1,000** |

注：DeepSeek V4-Flash 定价 $0.14/$0.28 每百万 token，成本显著低于 Anthropic。

### 4.3 盈利路径

- **第1年**：Web 版公开，积累用户和谣言数据库，打磨准确率
- **第2年**：启动付费版 + 企业 API，目标月收入 ¥10,000+
- **第3年**：规模化增长，目标年营收 ¥200 万+
