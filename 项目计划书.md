# 项目计划书 — AI 实时新闻真伪辨别平台「真相捕手」

---

## 一、项目概述

### 1.1 项目名称

**真相捕手（TruthCatcher）** —— AI 驱动的实时新闻真伪辨别平台

### 1.2 所属行业

人工智能 / 信息服务业 / 媒体科技（MediaTech）

### 1.3 项目背景

根据路透社《2025年数字新闻报告》，全球56%的受访者表示无法区分网上新闻的真假，虚假信息在社交媒体上的传播速度是真相的6倍。生成式AI的普及进一步降低了虚假内容的生产成本，"深度伪造"图片、视频、音频层出不穷。在中国，《中华人民共和国网络安全法》《网络信息内容生态治理规定》等法规明确要求平台承担信息内容审核责任，但现有的人工审核模式难以应对海量内容的实时甄别需求。

### 1.4 核心产品或服务

**一站式新闻真伪辨别SaaS平台**。用户通过网页端或浏览器插件输入新闻链接/文本，系统自动完成以下流程：

1. **全网信息采集**：调用多个搜索引擎并发检索相关报道，抓取正文内容
2. **信源可信度筛选**：对采集信息进行评级、交叉验证、虚假特征识别
3. **AI综合分析**：大语言模型基于筛选后的可靠信息进行推理判断
4. **结构化结果输出**：呈现来龙去脉、新闻摘要、真实性概率、正反论据、完整论证链

### 1.5 项目愿景

成为华语互联网用户最信赖的第三方新闻事实核查工具，让每个人都能便捷地了解自己读到的东西是真是假。

## 二、市场分析

### 2.1 目标用户画像

| 用户群体 | 占比估计 | 使用场景 | 核心诉求 |
| -------- | -------- | -------- | -------- |
| **普通网民** | 60% | 看到朋友圈/微博/抖音转发的惊人消息后，想验证真假 | 操作简单、结论直观、免费使用 |
| **内容创作者/自媒体** | 20% | 选题前核实信源，避免引用虚假信息导致声誉受损 | 来源可追溯、论据充分、可引用到自己的作品中 |
| **企业公关/风控部门** | 12% | 监测涉及本企业的舆情真伪，及时应对虚假负面信息 | 实时预警、批量监测、专业报告 |
| **学术/研究机构** | 5% | 研究虚假信息传播规律，分析典型案例 | 数据导出、传播路径可视化、API接口 |
| **政府/公共机构** | 3% | 舆情监控、谣言治理、公共信息发布前的核验 | 高准确率、安全合规、数据本地化 |

### 2.2 市场规模

- **全球Fact-Checking市场**：2024年约12亿美元，预计2030年达到38亿美元（CAGR 21%）
- **中国市场**：网络辟谣和信息核验相关市场规模2025年预计超过80亿元，涵盖平台内容审核（60亿）、企业舆情监测（15亿）、C端工具（5亿）
- **增长驱动**：AI内容生成爆发 → 虚假信息激增 → 核验需求同步增长

### 2.3 用户痛点

1. **信息过载**：每天被算法推送的"震惊体"包围，无力逐条验证
2. **缺乏专业工具**：现有辟谣平台多为定期汇总，无法实时检测
3. **信源难追溯**：营销号洗稿转述后，原始来源难以追踪
4. **立场干扰**：许多核验平台本身带有政治或商业立场，结论缺乏公信力
5. **跨境信息孤岛**：中文信息与英文/外文信息之间缺乏交叉验证渠道

### 2.4 竞争对手分析

| 竞争对手 | 类型 | 优势 | 劣势 |
| -------- | ---- | ---- | ---- |
| **澎湃·明查** | 人工核查团队 | 专业编辑团队、权威性强 | 每天只能核查3-5条，无法实时响应；仅覆盖热点事件 |
| **腾讯较真** | 平台自建 | 背靠腾讯生态、用户基数大 | 仅核实特定栏目内容，不开放给用户自主查询 |
| **FactCheck.org** | 国际人工核查 | 政治事实核查的标杆、非营利公信力 | 仅覆盖美国政治议题、英文为主 |
| **Snopes.com** | 国际人工核查 | 历史悠久（1994年）、覆盖都市传说/网络谣言 | 人工效率瓶颈、中文内容极少 |
| **NewsGuard** | 信源评级数据库 | 对7000+新闻网站进行人工评级、浏览器插件 | 仅评级信源，不分析具体文章内容 |
| **ChatGPT/Gemini** | 通用AI工具 | 即时响应、自然语言交互 | 非专用工具、可能编造信息、无系统化的网络检索验证流程 |

### 2.5 差异化竞争优势

1. **实时性**：从用户输入到输出结果，全流程自动化，目标响应时间<30秒；而人工核查团队需数小时到数天
2. **AI + 网络检索深度融合**：非简单问一句AI，而是自动执行多引擎检索 → 信源评级 → 交叉验证 → 综合推理的完整流水线
3. **中立透明**：专设公平性保障模块，正反论据均衡呈现，标注信源立场倾向和利益关联，不做"一槌定音"而是"让证据说话"
4. **中文优先 + 跨国验证**：深耕中文信息生态，同时自动检索英文信源做交叉验证，打破信息孤岛
5. **可追溯性**：每条论据附带来源链接，用户可自行核实，而非仅信任一个"AI评分"
6. **私人定制化**：用户可配置信源白名单/黑名单，调整可信度权重体系，适配不同使用场景

### 3.3 产品亮点（差异化卖点）

**亮点一：全自动核查流水线**

不同于人工核查团队每天处理3-5条新闻，也不同于通用AI的随口回答，真相捕手执行严格的多步骤核查流水线：检索 → 筛选 → 验证 → 分析 → 输出。每一步都可审计、可追溯。

**亮点二：信源评级透明化**

对每条采集到的信息标注来源类型和可信度等级：
- **A级**：官方政府网站、权威学术期刊
- **B级**：知名媒体、行业研究报告
- **C级**：中小媒体、行业博客
- **D级**：自媒体、论坛、匿名来源

用户可以一目了然地看到每条论据的可信程度，而非简单的AI结论。

**亮点三：强制正反均衡**

系统prompt要求必须同时搜集支持与反对新闻主张的证据，正反论据数量保持基本均衡。当一方明显缺乏证据时，系统会明确标注"当前未找到支持/反对的可靠证据"，而非直接判定真假。

**亮点四：立场声明与利益冲突标注**

当论据来源存在已知立场倾向（如某媒体属于特定政治阵营）或与新闻主体存在利益关联（如某研究由相关企业资助）时，系统会在论据卡片上明确标注，避免用户被表面权威误导。

**亮点五：完整的前因后果还原**

不仅判断真假，还帮助用户理解"为什么会这样"——这条新闻从哪来？谁最先发的？传播过程中发生了什么变化？相关事件背景是什么？

## 四、商业模式

### 4.1 收入模型总览

采用 **Freemium + 分层订阅 + B端服务** 的混合商业模式。

| 层级 | 定价（参考）| 目标用户 | 核心权益 |
| ---- | ----------- | -------- | -------- |
| **免费版** | ¥0/月 | 普通网民 | 每日3次分析、文本输入、基础结果展示 |
| **专业版** | ¥29/月 | 内容创作者、深度用户 | 每日50次分析、URL输入、论据溯源、历史记录、分析报告导出 |
| **企业版** | ¥299/月起 | 企业公关/风控部门 | 批量监测、关键词预警、API接口、团队协作、定制信源权重 |
| **机构定制版** | 议价 | 政府/学术机构 | 本地化部署、数据合规保障、专属模型微调、定制报告格式 |

### 4.2 收入来源详解

#### 来源一：C端订阅（预计占总收入40%）

- 免费版培养用户习惯，通过分享裂变获客
- 专业版面向内容创作者、媒体从业者等有高频核验需求的用户
- 预期转化率：免费→付费 3%-5%

#### 来源二：B端企业服务（预计占总收入45%）

- SaaS订阅：为企业提供舆情真伪监测平台，按监测关键词数量和频次阶梯定价
- API调用：按次计费，供其他平台/应用集成核验能力
- 定制报告：针对特定事件的深度分析报告，单份定价¥500-¥5000

#### 来源三：广告与流量变现（预计占总收入15%）

- 免费版在分析结果页展示非侵入式广告
- 不卖"首页推荐位"，避免影响中立形象
- 考虑"赞助真实性报告"模式：企业可赞助特定话题的核验报告，但系统仍保持分析独立性，赞助不影响结论

### 4.3 成本结构预估

| 成本项 | 月均预估（起步阶段）| 说明 |
| ------ | ------------------- | ---- |
| LLM API调用 | ¥3,000-8,000 | 按每日1000次分析估算（Claude/GPT API） |
| 搜索引擎API | ¥1,000-3,000 | SerpAPI/Bing API按次计费 |
| 云服务器 | ¥1,000-2,000 | 初期轻量部署 |
| 爬虫代理IP | ¥500-1,000 | 反反爬需要 |
| 域名/基础服务 | ¥200-500 | 域名、CDN等 |
| **合计** | **¥5,700-14,500** | 个人可负担的起步成本 |

### 4.4 盈利路径预估

- **第1年**：MVP上线，以免费用户积累为主，打磨产品体验和准确率，目标1万注册用户，收入以少量付费转化为主，预计月收入¥3,000-10,000
- **第2年**：产品成熟，启动B端推广，目标5万注册用户 + 20家企业客户，预计月收入¥30,000-80,000，实现盈亏平衡
- **第3年**：规模化增长，浏览器插件 + API开放 + 行业解决方案，目标年营收¥200万+

### 4.5 竞争优势护城河

1. **数据飞轮**：每次分析积累的信源评级数据、虚假特征数据会持续优化系统准确率，使用越多越准
2. **品牌信任**：坚持中立透明的核验理念，建立"最不偏不倚的事实核查工具"的公众认知
3. **技术壁垒**：核查流水线中的多引擎检索、信源评级、交叉验证、公平性约束等环节构成系统化壁垒，纯通用AI模型难以复制
4. **网络效应**：用户越多 → 数据越多 → 准确率越高 → 口碑越好 → 用户更多
